Note
 The completed code for this lab can be found at $COURSE_HOME/spring-cloud-config/complete/springbox-catalog.
In this lab, we’ll do the following

  • Create a GitHub repository to store our configuration

  • Create a Spring Cloud Config Server and test it with a basic sample application

  • Create Config Server Service in Cloud Foundry

  • Deploy the springbox-catalog to cloud foundry and use the Config Server

Now we’ll begin using the components found in Spring Cloud to implement patterns for distributed systems. We’ll being using Spring Cloud Config to centralize configuration for applications. Our configuration will be stored in a GitHub repository, and served to our applications using the Spring Cloud Config Server.

Approximate time: 20 minutes

Create Config Server Service in Cloud Foundry

  1. Go to Cloud Foundry UI https://apps.run.haas-80.pez.pivotal.io and create an instance of Config Server call it springbox-config-service . Do not bind it to any apps at this time.

  2. Click on the Service and Click on Manage

  3. Wait till the service has been created and you could see the configuration properties

  4. Update the service to point to your github uri for springbox-config-repo

    cf update-service springbox-config-service -c '{"git": { "uri": "https://github.com/regpaco/springbox-config-repo" } }'

  5. Verify that the changes have been updated

Deploy the springbox-catalog to cloud foundry and use the Config Server

  1. Change to the directory:

    $ cd $COURSE_HOME/springcloud-config/initial/springbox-catalog

  2. Modify src/main/resources/bootstrap.yml and into that file paste the following (provide the matching application name!):

    spring:
  application:
    name: springbox-catalog
  cloud:
    config:
      uri: ${vcap.services.springbox-config-service.credentials.uri:http://localhost:8888}
    Note
    		 This step is needed because of security is enabled on CF by default

  3. Modify src/main/resources/application.yml

    management:
  security:
    enabled: false
security:
  basic:
    enabled: false

  4. Add the springbox-config-service service to the manifest.yml:

    ---
applications:
- name: springbox-catalog
  host: springbox-catalog-${random-word}
  memory: 512M
  instances: 1
  path: target/springbox-catalog-0.0.1-SNAPSHOT.jar
  env:
    SPRING_PROFILES_ACTIVE: cloud
    TRUST_CERTS: api.run.haas-80.pez.pivotal.io  # <--- Add this!
  services:
    - springbox-catalog-db
    - springbox-config-service  # <--- Add this!

  5. Open pom.xml, add the dependency management:

      <dependencyManagement>
    <dependencies>
    ...
      <dependency>
        <groupId>io.pivotal.spring.cloud</groupId>
        <artifactId>spring-cloud-services-dependencies</artifactId>
        <version>1.3.0.RELEASE</version>
        <type>pom</type>
        <scope>import</scope>
      </dependency>
      ...
    </dependencies>
  </dependencyManagement>

  6. And add the appropriate dependencies for the Spring Cloud Services Config:

        <dependency>
      <groupId>io.pivotal.spring.cloud</groupId>
      <artifactId>spring-cloud-services-starter-config-client</artifactId>
    </dependency>

  7. Build the JAR:

    $ mvn package

  8. Push the application:

    $ cf push
