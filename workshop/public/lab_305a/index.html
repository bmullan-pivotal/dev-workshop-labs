Note
 The completed code for this lab can be found at $COURSE_HOME/spring-cloud-eureka/final/springbox-catalog.
In this lab, we’ll do the following

  • Create Spring Cloud Service Registry in Cloud Foundry

  • Deploy springbox-catalog in Cloud Foundry

  • Deploy springbox-client in Cloud Foundry

Approximate time: 20 minutes

Exercises

Create Spring Cloud Service Registry in Cloud Foundry

  1. Go to Cloud Foundry UI https://apps.run.haas-80.pez.pivotal.io and add an instance of Service Registry from the marketplace

  2. Call it springbox-service-registry . Do not bind it to any apps at this time.

  3. Click on the Service and Click on Manage

  4. Wait till the service has been created and you could see the configuration properties

Deploy springbox-catalog in Cloud Foundry

  1. Change to the lab directory:

    $ cd $COURSE_HOME/springbox-catalog

  2. Create manifest.yml file under springbox-catalog and copy the following contents:

    ---
applications:
- name: springbox-catalog
  host: springbox-catalog-${random-word}
  memory: 1G
  instances: 1
  path: target/springbox-catalog-0.0.1-SNAPSHOT.jar
  env:
    SPRING_PROFILES_ACTIVE: cloud
    TRUST_CERTS: api.run.haas-80.pez.pivotal.io  # <--- Add this!
  services:
  - springbox-catalog-db
  - springbox-config-service
  - springbox-service-registry  # <--- Add this!
  timeout: 120

  3. Open pom.xml, and add the appropriate dependencies for the Spring Cloud Services Config:

        <dependency>
      <groupId>io.pivotal.spring.cloud</groupId>
      <artifactId>spring-cloud-services-starter-service-registry</artifactId>
    </dependency>

  4. Build the JAR:

    $ mvn package

  5. Push the application:

    →cf push

  6. Click on the Service and Click on Manage

  7. You should see the service registered

Deploy springbox-client in Cloud Foundry

  1. Change to the lab directory:

    $ cd $COURSE_HOME/springbox-client

  2. Create manifest.yml file under springbox-client and copy the following contents:

    ---
applications:
- name: springbox-client
  host: springbox-client-${random-word}
  memory: 1G
  instances: 1
  path: target/springbox-client-0.0.1-SNAPSHOT.jar
  env:
    SPRING_PROFILES_ACTIVE: cloud
    TRUST_CERTS: api.run.haas-80.pez.pivotal.io  # <--- Add this!
  services:
  - springbox-config-service
  - springbox-service-registry  # <--- Add this!
  timeout: 120

  3. Open pom.xml, add the dependency management:

      <dependencyManagement>
    <dependencies>
    ...
      <dependency>
        <groupId>io.pivotal.spring.cloud</groupId>
        <artifactId>spring-cloud-services-dependencies</artifactId>
        <version>1.3.0.RELEASE</version>
        <type>pom</type>
        <scope>import</scope>
      </dependency>
      ...
    </dependencies>
  </dependencyManagement>

  4. And add the appropriate dependencies for the Spring Cloud Services:

        <dependency>
      <groupId>io.pivotal.spring.cloud</groupId>
      <artifactId>spring-cloud-services-starter-config-client</artifactId>
    </dependency>
    <dependency>
      <groupId>io.pivotal.spring.cloud</groupId>
      <artifactId>spring-cloud-services-starter-service-registry</artifactId>
    </dependency>

  5. Build the JAR:

    $ mvn package

  6. Push the application:

    →cf push

  8. You should see the service registered
